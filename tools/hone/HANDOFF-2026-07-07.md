## 1. What this is (in one paragraph)

`hone` (at `~/code/minnows/tools/hone`, branch `hone/workflow-substrate`, worktree
`~/.tmp/minnows-substrate`) is a repo-quality engine: it plans refactoring/test work, has a code-writing
agent (the "maker") produce a change, proves the change with **deterministic gates** (baseline+post
evidence rungs: tests, complexity remeasure, scope/touchset containment, portability), and — for
non-certified classes — has a **different-provider model judge** review before landing. Books
(claims/cost ledgers, receipts, packet statuses) are the machine's durable memory. The terminal goal is
the PDPP reference-implementation at ~100 on the SLVP quality bar: every owned subsystem internally-pure /
product-pure / rented-pure / deleted / justified, sequenced by attention-weighted leverage.

## 2. The load-bearing design rules (violate these and it breaks)

- **maker ≠ judge (canon R3).** The producer of a change never certifies it. A *deterministic* check
  (script) is NOT the judge and may be run by anyone — but a *model* judgment must be a different provider
  than the maker. codex maker → claude judge, or vice-versa. `--maker codex --judge codex` REFUSES
  (collision). This is why the deferred-judge mode below exists.
- **Deterministic gates are never deferred or softened.** You may defer the *model judge* (see §5), never
  the rungs. A preflight/gate may DISCARD a malformed order but must NEVER weaken a rung to admit it —
  that's the maker lowering its own bar.
- **Behavior-preservation is a gate, not a hope.** Extractions must be proven against the REAL test suite
  (with the right backend/DB), not just "looks equivalent."
- **RED-tier is owner-gated, always.** Any function whose ~200-line neighborhood mentions grant.streams,
  streamGrant.fields, field_not_granted, access_mode, redirect_uri, forbidden-ip, token/consent/bearer/
  scope-enforcement, startServer/boot → the machine must SKIP it. These need a ratified owner campaign, not
  autonomous work. A wrong move there is a data-authorization bug, not a style nit.

## 6. Non-obvious traps that cost me hours (avoid these)

- **pgrep self-match:** an inline `sh -c 'while pgrep -f "hone work"...'` matches ITSELF → deadlocks silently.
  Put wait-loops in a script FILE (ps shows the path, not the contents). Bit me 3×.
- **Verify commits by SHA, never by grepping hook output.** The complexity gate silently rejected a commit
  while I "verified" the working tree; the commit hadn't happened. `git log --oneline -1` is truth.
- **biome writes diagnostics to STDERR** — always `2>&1` when counting complexity, or you get false zeros.
- **Measure complexity by MASS (Σ excess-cc), not COUNT.** Decompositions inflate the count (splitting one
  hairball raises the flagged-fn count). `grep "complexity of N" | sum(N-5)` is the honest metric.
- **Fresh worktrees need node_modules hardlinked at ALL layers** (root, packages/*, apps/*) or tests fail
  with `next: not found` / `Cannot find module 'ajv'` — env-only reds that look like real failures.
- **Test rungs MUST use per-file ephemeral Postgres DBs** (`pdpp_hone_*` + `--test-force-exit`), never the
  shared `pdpp` DB — shared-DB state pollution + scheduler-interval hangs = 30-min timeouts. The rung-builder
  now enforces this; hand-authored rungs must too.
- **Clawmeter is EST-first.** "OpenAI 95% 7d" ≠ exhausted if it resets soon + reset-credits are banked.
  "Claude 7d 120%-est" IS the ration target even at 82% actual. Read ests + resets + credits, not raw %.
  The `--defer-judge` mode exists precisely so a drain spends ZERO Claude when 7d is over ceiling.
- **Agent-tool model override silently spawned Fable once** (burned quota). Delegate heavy work via the
  Workflow tool (model:'opus' verified real in journal) or codex exec, not raw Agent spawns, until you've
  confirmed the model.
- **Crash recovery is idempotent:** revert non-quality residue, `rm .locks/*`, reset in_progress packets.
  The pattern is safe to run any time; it only touches this op's worktree.
